# TodoList TDD — 用户故事 (User Journeys)

## 用户故事

### Story 1: 添加待办
**角色**: 作为用户
**动作**: 我想添加一个新的待办事项
**收益**: 这样我可以记录需要完成的任务

**验收标准**:
- [ ] 输入框可以输入文本
- [ ] 按 Enter 或点击按钮可以添加待办
- [ ] 添加后输入框自动清空
- [ ] 新待办显示在列表顶部
- [ ] 空内容不可添加

### Story 2: 查看待办列表
**角色**: 作为用户
**动作**: 我想查看所有待办事项
**收益**: 这样我可以了解还有多少任务需要完成

**验收标准**:
- [ ] 显示所有待办项（内容 + 完成状态）
- [ ] 待办总数实时更新
- [ ] 列表为空时显示空状态提示

### Story 3: 切换完成状态
**角色**: 作为用户
**动作**: 我想切换待办的完成状态
**收益**: 这样我可以标记任务完成或撤销完成

**验收标准**:
- [ ] 点击复选框或文字可以切换状态
- [ ] 已完成项显示删除线样式
- [ ] 已完成项默认折叠（可选）

### Story 4: 删除待办
**角色**: 作为用户
**动作**: 我想删除一个待办事项
**收益**: 这样我可以移除不需要的任务

**验收标准**:
- [ ] 点击删除按钮可以删除
- [ ] 删除时无确认（简单场景）
- [ ] 删除后列表自动更新

### Story 5: 筛选待办
**角色**: 作为用户
**动作**: 我想按状态筛选待办
**收益**: 这样我可以只看进行中或已完成的

**验收标准**:
- [ ] 三个筛选按钮：全部 / 进行中 / 已完成
- [ ] 当前筛选按钮高亮
- [ ] 切换筛选后列表正确更新

### Story 6: 清除已完成
**角色**: 作为用户
**动作**: 我想一键清除所有已完成的待办
**收益**: 这样我可以快速清理已完成的任务

**验收标准**:
- [ ] 存在已完成后才显示清除按钮
- [ ] 点击后删除所有已完成项
- [ ] 列表自动更新
